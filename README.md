# 钢铁行业产业链碳足迹数据服务系统（前端原型）

本仓库为「钢铁行业产业链碳足迹数据服务系统」的 **前端原型工程**，用于产品方案评审、交互演示和需求对齐。  
当前形态为 **多页面静态站点**，基于 **Vue 3** 与 **Element Plus**（CDN 引入），涵盖 **运营管理端**、**供应商工作台**、**认证机构工作台** 等角色视角。

---

## 业务概览

系统支持钢铁产业链碳足迹 **核算与认证** 全流程：商城下单 → 运营配置任务与模板 → 供应商填报数据与凭证 → **LCA 计算**（独立系统，当前手工摆渡）→ 第三方核查 → 证书签发与 **报告下发** → **供应商确认接收**（或申诉）→ **待归档 / 已归档**，形成 **订单 → 任务 → 报告** 的闭环。

- **任务阶段（7 段）**：配置 → 采集（含审核）→ 计算 → 核查 → **下发** → **待归档** → **已归档**
- **角色**：L1 商城用户、L2 供应商、L3 运营人员、L4 核查机构；原型覆盖运营端、供应商端、认证机构端
- **关键规则**：运营下发报告/证书后，供应商在「我的报告」中可 **确认接收** 或 **提起申诉**；仅确认接收后运营可执行待归档/归档；待归档与已归档下不展示申诉处理按钮。详见 `docs/01_整体业务与产品设计审查.md` 与 `docs/04_任务调度与状态机.md`

## 本地开发与预览

### 环境要求

- **Node.js**：建议 ≥ 18.x  
- **包管理**：默认 `npm`（可改用 pnpm / yarn）

### 安装依赖

```bash
npm install
```

### 启动开发服务器

```bash
npm run dev
```

浏览器访问提示的本地地址，例如：

- 门户：`http://localhost:5173/`
- 运营端自营任务列表：`http://localhost:5173/operator/self_operated_task_list.html`
- 供应商报告：`http://localhost:5173/supplier/reports.html`

### 构建与预览生产版本

```bash
npm run build
npm run preview
```

- 产物输出到 `dist/`（多页面 HTML、CSS 及 public 下静态资源）
- 部署：将 `dist/` 整体上传至静态托管（Nginx、对象存储、GitHub Pages 等）；若部署在子路径，请在 `vite.config.js` 中设置 `base: '/你的子路径/'`

## 当前实现要点

- **统一布局与 Vue 启动**：运营端通过 `runOperatorApp(component)`（`public/js/layout.js`）统一完成 `createApp`、Element Plus + 图标、`mount('#app')`；公共基础样式在 `css/common.css`（全高布局、`[v-cloak]` 等），各页脚本精简。
- **任务 7 段与下发**：自营/委托任务列表展示 配置 / 采集 / 计算 / 核查 / **下发** / 待归档 / 已归档；报告管理支持勾选报告/证书执行「下发」至供应商。
- **报告详情与申诉**：运营端 `report_detail.html` 查看报告；待归档/已归档不展示申诉按钮。供应商端「我的报告」支持预览、**确认接收**与**申诉**窗口（仅下发后、确认接收前可申诉）。
- **Mock 数据**：`js/mockOperator.js`、`mockTasks.js` 及 `docs/07_Mock数据说明.md` 约定订单号/任务号规则，便于与后端接口对接时替换。
