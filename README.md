# 钢铁行业产业链碳足迹数据服务系统（前端原型）

本仓库为「钢铁行业产业链碳足迹数据服务系统」的 **前端原型工程**，用于产品方案评审、交互演示和需求对齐。  
当前形态为 **多页面静态站点**，基于 **Vue 3** 与 **Element Plus**（CDN 引入），涵盖 **运营管理端**、**供应商工作台**、**认证机构工作台** 等角色视角。**当前原型为演示用途**：供应商/认证机构工作台未强制登录即可访问，正式环境需接入身份与权限。

---

## 业务概览

系统支持钢铁产业链碳足迹 **核算与认证** 全流程：商城下单 → 运营配置任务与模板 → 供应商填报数据与凭证 → **LCA 计算**（独立系统，当前手工摆渡）→ 第三方核查 → 证书签发与 **报告下发** → **供应商确认接收**（或申诉）→ **待归档 / 已归档**，形成 **订单 → 任务 → 报告** 的闭环。

- **任务阶段（5 段）**：**0 配置** → **1 采集**（含审核）→ **2 计算** → **3 核查** → **4 报告**。阶段 4「报告」在列表可展示子状态（待处理 / 已下发 / 待归档 / 已归档 / 已作废），所有报告相关操作统一在报告管理完成。
- **角色**：L1 商城用户、L2 供应商、L3 运营人员、L4 核查机构；原型覆盖运营端、供应商端、认证机构端。**当前版本不包含 L1 商城侧**：订单由运营端 Mock 数据或对接系统生成，运营在「订单管理」中可见并执行「确认并分配任务」；正式环境订单来源为商城下单与支付回调。**L4 链主企业、钢铁客户**工作台为规划占位，后续迭代开放，用于链主采购/绿色供应商库与客户验真/碳标签等场景。
- **关键规则**：运营下发报告/证书后，供应商在「我的报告」中可 **确认接收** 或 **提起申诉**；仅确认接收后运营可执行待归档/归档；待归档与已归档下不展示申诉处理按钮。详见 `docs/01_业务与流程/01_整体业务与产品设计审查.md` 与 `docs/01_业务与流程/04_任务调度与状态机.md`

## 技术栈与工程化

- **前端框架**：Vue 3（全局构建版本，CDN 引入）。主门户 `index.html` 为静态页；Vue 用于 `pcf.html` 及运营/供应商/认证机构工作台等页面。
- **UI 组件库**：Element Plus（CDN 引入）
- **表格引擎**：SpreadJS 16.x（模板与填报表统一方案，见 `docs/02_功能与对接/05_模板引擎解析逻辑.md`）
- **构建与开发服务器**：Vite
- **语言**：原型阶段使用原生 JavaScript 与 HTML/CSS

### 门户与产品碳足迹页（index / pcf）

- **`index.html` — 系统门户页**
  - **定位**：钢铁行业产业链可信数据空间的统一入口，不直接做角色跳转；运营/供应商/认证机构等工作台入口在二级应用门户 `pcf.html` 及各自页面内。
  - **主要内容**：唯一核心区块为 **「平台应用与产品服务」**，基于可信数据空间的数据应用与数据产品规划，以卡片形式列出共 **10 项** 服务（数据应用 + 数据产品）。当前以 **「产品碳足迹核算与核查闭环服务」** 为首发应用并标记为「已上线」，点击该卡片进入二级门户 `pcf.html`；其余 9 项为规划中，随版本迭代与数据空间上架逐步开放。
  - **10 项服务概览**：① 产品碳足迹核算与核查闭环服务（已上线，进 pcf）② 供应商碳数据采集与协同服务 ③ 第三方核查与碳数据存证审计支持 ④ 行业环境产品声明构建与应用支持服务 ⑤ 气候情景分析与风险评估支持服务 ⑥ 绿钢数据库与行业动态 ⑦ 绿色金融与转型金融支持服务 ⑧ 碳边境调节机制与数字产品护照生成支持 ⑨ 环境、社会与治理披露报告数据支撑服务 ⑩ 供应商评价体系建设支持服务。
  - **页脚**：运营方为鑫智链（南京鑫智链科技信息有限公司），版本号等。

- **`pcf.html` — 产品碳足迹应用门户**
  - **定位**：从 index 点击「产品碳足迹核算与核查闭环服务」后进入的二级门户，承担方案介绍、生态角色说明与 **工作台入口**（运营端、供应商工作台、认证机构工作台）。
  - **面包屑**：鑫智链双碳运营平台 › 产品碳足迹认证服务，可返回 `index.html`。
  - **章节结构**（共 6 大块）：
    1. **挑战**：钢铁供应链的「碳数据困境」——能源数据未自动化、数据获取成本高、协同效率低（含三张数据卡片）。
    2. **方案：鑫智链「1+4」可信生态**：生态系统角色图与工作台入口。**L3** 为「产品碳足迹数据服务系统」（生态运营者 | 生命周期评价计算联合中心 | 信任规则定义者），对应功能为生态伙伴与应用管理、碳足迹核算任务调度、模板与标准中心，点击整块可 **进入运营管理端**。下方为 **L4** 四张卡片：**供应商**（可信数据生产者，进入供应商工作台）、**第三方认证机构**（可信价值公证者，如 SGS/DNV 等，进入认证机构工作台）、**链主企业**（可信价值应用者，工作台占位「敬请期待」）、**钢铁客户**（可信价值验证者，工作台占位「敬请期待」）。
    3. **核心流程**：从数据到可信资产的 5 阶段（服务开通 → 联合计算 → 可信认证 → 报告与凭证 → 价值应用），含流程图式卡片。
    4. **数据洞察：数据质量与协同复验**：说明原始数据与协同复验的重要性，配两张图：典型任务中数据来源类型占比（认证/已复验、自声明、估算）、认证数据对碳足迹结果的影响。
    5. **TDS 技术：构建「信任」的四大支柱**：按 PDRD 表述——① 身份可信（DID）② 流程可信（数字合约）③ 使用可信（使用控制，如 SGS 沙箱核验）④ 结果可信（VC 可验证凭证）。
    6. **价值实现**：从认证到资产应用（钢铁供应链联合碳排放分析、碳足迹凭证在鑫采商城/绿色招标/绿色金融等场景的应用）。
  - **入口跳转**：点击 L3 区块或「供应商工作台」「认证机构工作台」卡片上的「进入系统 →」分别跳转至 `operator/dashboard.html`、`supplier/dashboard.html`、`certifier/dashboard.html`。

## 本地开发与预览

### 环境要求

- **Node.js**：建议 ≥ 18.x  
- **包管理**：默认 `npm`（可改用 pnpm / yarn）

### 安装依赖

```bash
npm install
```

### 启动开发服务器

```bash
npm run dev
```

浏览器访问提示的本地地址，例如：

- 门户：`http://localhost:5173/`
- 运营端自营任务列表：`http://localhost:5173/operator/self_operated_task_list.html`
- 供应商报告：`http://localhost:5173/supplier/reports.html`

### 构建与预览生产版本

```bash
npm run build
npm run preview
```

- 产物输出到 `dist/`（多页面 HTML、CSS 及 public 下静态资源）
- 部署：将 `dist/` 整体上传至静态托管（Nginx、对象存储、GitHub Pages 等）；若部署在子路径，请在 `vite.config.js` 中设置 `base: '/你的子路径/'`

## 当前实现要点

- **统一布局与 Vue 启动**：运营端通过 `runOperatorApp(component)`（`public/js/layout.js`）统一完成 `createApp`、Element Plus + 图标、`mount('#app')`；公共基础样式在 `css/common.css`（全高布局、`[v-cloak]` 等），各页脚本精简。
- **SpreadJS 模板与填报表**：模板详情、任务配置、供应商填报、采集审核等页统一使用 SpreadJS 16.x 展示模板/采集表；`js/spreadUtils.js` 提供 Snapshot 加载/导出、可编辑列、只读控制；凭证与工作簿绑定，无行级关联。详见 `docs/02_功能与对接/05_模板引擎解析逻辑.md`。
- **任务 5 段与报告**：自营/委托任务列表展示 配置 / 采集 / 计算 / 核查 / **报告**（阶段 4 可带子状态：待处理/已下发/待归档/已归档/已作废）；阶段 4 点击跳转报告管理，报告管理支持勾选报告/证书执行「下发」至供应商。
- **报告详情与申诉**：运营端 `report_detail.html` 查看报告；待归档/已归档不展示申诉按钮。供应商端「我的报告」支持预览、**确认接收**与**申诉**窗口（仅下发后、确认接收前可申诉）。
- **Mock 数据**：`js/mockOperator.js`、`mockTasks.js` 及 `docs/02_功能与对接/07_Mock数据说明.md` 约定订单号/任务号/Snapshot Mock 规则，便于与后端接口对接时替换。
