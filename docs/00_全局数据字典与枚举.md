# 全局数据字典与枚举定义

## 1. 核心状态机 (State Machine)

### 1.1 订单状态 (Order Status)
| 状态值 | 描述 | 触发条件 |
| :--- | :--- | :--- |
| `UNPAID` | 待支付 | 订单创建初始状态 |
| `PAID` | 已支付 | 支付回调成功 |
| `CONFIRMED` | 已确认 | 运营人员介入或系统自动确认 |
| `COMPLETED` | 已完成 | 所有关联子任务均结束 |

### 1.2 任务状态 (Task Status) - **核心**
此状态流转应用于 `task_detail.html` 页面。
1. **`CONFIG` (待配置)**: 任务生成，等待运营人员选定模板。
2. **`COLLECT` (待填报)**: 模板下发，等待供应商提交数据。
3. **`AUDIT` (待审核)**: 供应商提交，运营人员检查数据/凭证完整性。
4. **`CALC` (计算中)**: 数据无误，调用 LCA 引擎计算。
5. **`VERIFY` (核查中)**: 报告生成，推送到 SGS/DNV 系统。
6. **`CERTIFIED` (已发证)**: 核查通过，生成 VC 证书并上链。

## 2. 角色权限 (Roles)
* **L1 商城用户**: 下单、支付、签署商务协议。
* **L2 供应商 (Supplier)**: 在供应商工作台填报数据、上传凭证。
* **L3 运营人员 (Operator)**: 配置模板、审核数据、调度任务。
* **L4 核查机构 (Verifier)**: 查看穿透数据、反馈核查意见。
