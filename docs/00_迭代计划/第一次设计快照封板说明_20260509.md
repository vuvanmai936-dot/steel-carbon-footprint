# 第一次设计快照封板说明（全仓）

## 1. 封板目的

将 **2026-05-09** 当日仓库内**已纳入 Git 的全部路径**（见 §2）固化为 **产品经理设计交付的第一次快照（v1）**，作为需求/设计评审、客户对照与争议追溯的**只读基线**。  
**不替代**后续正式开发环境中的运行态与接口实现；引入开发环境后，**后续设计以开发环境所约定的仓库/分支/提交为事实源（SoT）**，本快照仅作 v1 对照。

## 2. 封板范围

以下目录与文件（以本仓库根为起点，且受 `.gitignore` 排除者除外）均属 v1 设计快照范围：

- `docs/`：业务、流程、功能、任务详情、澄清与消息、标准与合规及迭代计划等设计文档  
- `operator/`、`supplier/`、`certifier/`：多页原型 HTML  
- `js/`、`css/`、`public/`：布局、Mock、SpreadJS 工具与静态资源  
- 根目录：`index.html`、`pcf.html`、`vite.config.js`、`package.json`、`README.md`、`AGENTS.md` 等工程与入口说明  
- `.cursor/rules/`：Cursor 项目规则（团队与 AI 协作约定）  
- `客户商务/`：商务与建设方案相关材料（**对外分发须单独做脱敏与授权**）  

**不在 Git 对象内、但影响本地/演示运行的依赖**（如 SpreadJS、Vue、Element Plus 的 CDN 引用）以当时 [`README.md`](../../README.md) 与 [`docs/02_功能与对接/05_模板引擎解析逻辑.md`](../02_功能与对接/05_模板引擎解析逻辑.md) 的说明为准；封板不锁定 CDN 远端版本号。

## 3. 时点与 Git 对象

| 项 | 值 |
|----|-----|
| 封板日历日 | 2026-05-09 |
| 附注标签（annotated tag） | `design-freeze-v1-20260509` |
| 标签含义 | 指向「全仓内容 + 本封板说明」合并后的**单一提交**，即 v1 设计快照在 Git 中的可复现锚点 |

**查验（任一克隆本仓库的机器上）**：

```bash
git fetch --tags
git show design-freeze-v1-20260509 --no-patch
git rev-parse design-freeze-v1-20260509^{commit}
```

上述 `git rev-parse` 的输出即本快照对应的 **40 位提交哈希**（不在本文档内嵌该哈希，避免「文档自指提交」在 amend 时产生震荡；以 Git 标签解析结果为准）。

**可选只读分支**（与标签**不得同名**，避免 `refname is ambiguous`）：例如 `git branch snapshot/design-freeze-v1-20260509 design-freeze-v1-20260509`，便于 `git diff`。

## 4. 权威性与用途

- **业务规则与状态机**仍以 `docs/01_业务与流程`、`docs/02_功能与对接` 等目录下既有文档表述为准；本封板**不新增**业务规则，仅锁定**该时点的版本组合**。  
- 用于：与客户的「当时交付界面与文档」对齐、内部迭代对比、报价/范围回溯。  
- 若需离线包，可自行对标签检出目录打 zip，并在内部台账记录 zip 的 SHA256（非本仓库必选交付物）。
