# 任务执行工作台 (Task Workspace) 逻辑说明

## 1. 页面概述
此页面是系统的核心生产车间，采用**单页应用 (SPA)** 模式，基于 `activeStep` 变量控制视图渲染。

## 2. 阶段详细逻辑

### Stage 1: 配置 (Config)
* **前置条件**: 任务状态为 `CONFIG`。
* **数据来源**: 调用 `模板中心` 接口，拉取状态为 `Active` 的模板列表。
* **动作**: 运营人员选择模板后，系统需将模板结构（Sheet/Items）复制一份到当前任务实例中（Snapshot），避免后续模板变更影响进行中的任务。

### Stage 2: 采集与审核 (Collect & Audit)
* **视图逻辑**:
  * 左侧展示 Excel 结构（Tabs 分组）。
  * 右侧展示对应工序的“凭证要求”。
* **审核动作**:
  * **驳回**: 需弹窗填写驳回原因，任务状态回退至 `COLLECT`（供应商需重填）。
  * **通过**: 任务状态流转至 `CALC`。

### Stage 3: 三方核查 (Interface 4.4)
* **交互逻辑**: 
  * 这是一个**异步等待**过程。
  * 右侧“协作日志”需实时轮询（或 WebSocket）三方系统的消息。
  * 若收到 SGS 的“澄清请求”，需高亮显示并在日志中弹窗提醒。

## 3. 协作日志 (Log System)
日志需记录以下维度的操作：
* **时间**: 精确到秒。
* **操作人**: 系统 / 运营 / 供应商 / 核查员。
* **动作**: 状态变更、文件上传、澄清发起。
