# 供应商工作台功能清单与信息结构

本文档描述供应商工作台（L2 角色）的功能清单、信息结构及与运营端的对应关系。技术栈与运营端统一：Vue 3、Element Plus、侧栏+顶栏+主区布局，详见 [js/supplierLayout.js](../js/supplierLayout.js)。

## 一、功能清单

| 序号 | 功能模块 | 说明 | 对应页面 | 优先级 |
|------|----------|------|----------|--------|
| 1 | 工作台 | 概览统计（待办数量、待澄清数量、报告数）、快捷入口（待办任务、我的报告、返回门户） | dashboard.html | P0 |
| 2 | 待办任务 | 列表展示任务（任务编号、订单号、产品、规格、类型/状态、最新通知）；类型：待填报、采集已驳回、待澄清/待整改；操作：填报 / 处理（跳填报页或澄清页） | task_list.html | P0 |
| 3 | 任务填报 | 产品与订单只读、工序数据填报与凭证上传、驳回原因展示（采集已驳回时）、提交填报 | task_fill.html | P0 |
| 4 | 待澄清处理 | 任务与产品只读、澄清/驳回说明展示、回复说明与佐证上传、提交回复 | task_clarify.html | P0 |
| 5 | 我的报告 | 报告仅在**运营下发后**出现在列表；每条报告：**预览**、**确认接收**、**对报告有异议（申诉）**；**确认接收后该报告不可再申诉**。交互：预览后选择「确认接收」或「不接收 → 填写原因/附件 → 提交申诉」 | reports.html | P0 |
| 6 | 企业信息与证照（身份认证） | 企业基本信息、联系人、证照管理（营业执照必选，ISO 等可选）；支持应用内维护并「提交至空间」或「从空间同步」展示 | identity.html | P0 |
| 7 | 市场 | 可选入口（当前为占位） | market.html | P2 |

说明：P0 为与运营端协同的主流程（待办→填报/澄清→报告→申诉）及**企业信息与证照**（无此则无法向空间提交数据，运营/核查端无法获取营业执照等）；P2 可保留占位，后续迭代。

## 二、信息结构

### 2.1 菜单与路由

- **一级菜单**：工作台、待办任务、我的订单、我的报告、企业信息、绿色生态市场、知识库与帮助（企业信息为实际功能，市场/帮助可为占位）。
- **无二级菜单**：供应商端功能扁平。
- **详情/子页**：填报、澄清为列表跳转页（task_fill.html?taskId=、task_clarify.html?taskId=），不列入侧栏菜单。

### 2.2 核心数据实体（与 02/04 及 Mock 一致）

| 实体 | 主要字段 | 说明 |
|------|----------|------|
| 待办任务 | taskId, orderNo, productName, specs, taskType, rejectReason?, clarificationText? | taskType: pending_fill / rejected_collect / clarification；与 js/mockTasks.js 及运营端任务一致 |
| 任务详情（填报/澄清） | 继承 MOCK_TASK_MAP：taskId, orderNo, productName, specs, supplier | 填报页可选 rejectReason；澄清页 clarificationText 按 taskId Mock |
| 报告 | taskId, productName, reportStatus, reportDate, received? | 报告仅在运营下发后出现；reportStatus：已出报告/已发证；received：是否已确认接收，确认接收后不可申诉 |
| 申诉 | taskId, productName, reason, attachments? | 仅未确认接收时可提交；提交后 Mock 提示，运营端 report_mgt.html 待处理申诉区展示 |

### 2.3 与运营端的数据对应关系

- 任务 ID（taskId）、订单号（orderNo）与运营端 **js/mockTasks.js**（MOCK_TASK_MAP）、订单 Mock、自营/委托任务列表一致；报告与 **js/mockOperator.js**（MOCK_REPORTS）及 report_mgt 下发记录一致，taskNo = taskId，便于联调与演示。
- 供应商「待澄清」与运营端 task_detail_lca.html「转派给供应商」、task_detail_collect.html 驳回形成闭环；报告与 report_mgt.html 的档案与申诉处理对应。报告仅在运营下发后可见；供应商「确认接收」后该报告不可再申诉，与 04 下发/归档规则一致。
- **推荐演示路径**（运营下发 → 供应商可见）：运营端报告管理对某任务执行「下发」后，供应商端「我的报告」中应出现对应 taskId 的报告；演示时请使用与 07_Mock数据说明 中「7.2 推荐演示路径」一致的订单号/任务号，以保证闭环可走通。

### 2.4 供应商信息与证照数据流（TDS/接入连接器）

企业信息与证照是向可信数据空间提交、并供运营端与核查端使用的基础数据。支持两种数据路径（可择一或兼容）：

- **路径 A（注册空间时已上传）**：供应商在注册/接入可信数据空间时，在空间侧上传企业信息与证照；空间/接入连接器将数据同步到应用。应用内以**只读展示**为主，提供「从空间同步」按钮及「申请变更」入口。
- **路径 B（应用内维护）**：供应商在应用内维护企业基本信息、联系人、证照（营业执照必选，其它资质可选），通过「保存」「提交至空间」提交；由接入连接器写入可信数据空间，运营端、核查端从空间获取的数据即为此来源。当前原型优先支持路径 B，提交后 Mock 提示「已提交至可信数据空间」。

证照清单与中心端供应商详情「资质证书」一致：**营业执照（副本）** 必填；可选如 ISO 14001 等，来源可标注「本地上传」或「空间同步」。

## 三、技术栈与布局

- **脚本**：Vue 3、Element Plus、@element-plus/icons-vue、element-plus 中文 locale、supplierLayout.js（SupplierLayout.setup、runSupplierApp）。
- **样式**：common.css、operator.css（复用布局与表格）、supplier.css（主题色/激活色）。
- **布局**：el-container > el-aside(220/64) + el-container > el-header(60px) + el-main；侧栏标题「供应商工作台」，菜单 active 色 #059669；头部面包屑 + 头像 + **返回门户**（统一指向 `pcf.html` 产品碳足迹应用门户，与认证机构端一致）。
