# 核查机构端功能清单与入口说明

本文档描述核查机构端（L4 角色）的功能清单与入口。技术栈与供应商端一致：Vue 3、Element Plus、侧栏+顶栏+主区布局，详见 [js/certifierLayout.js](../js/certifierLayout.js)。

## 一、入口

- **门户入口**： [pcf.html](../pcf.html) 产品碳足迹认证服务页 → 点击「［L4］ 第三方认证机构」卡片，跳转至 **certifier/dashboard.html**（核查仪表盘）。
- **可选登录页**：certifier/login.html 为**演示用**登录（U-Key / 账号密码），登录成功后跳转 dashboard.html。**当前原型为演示用途**：认证机构工作台未强制登录即可访问；正式环境需接入身份与权限，登录页仅作交互演示。身份登记与接入的架构说明见 [22_TDS 架构与合规设计说明](../05_标准与合规/22_TDS_架构与合规设计说明.md)。

## 二、功能清单

| 序号 | 功能模块 | 说明 | 对应页面 | 优先级 |
|------|----------|------|----------|--------|

| 2 | 核查任务列表 | 待核查任务列表，穿透查看报告与凭证，操作：核查（跳详情） | task_list.html | P0 |

## 三、菜单与路由

- **一级菜单**：核查仪表盘、核查任务列表、证书颁发管理、结算中心、机构管理。
- **详情页**：任务详情为列表跳转（task_detail.html?taskId=xxx），侧栏高亮「核查任务列表」。
- **布局**：el-container > el-aside(220/64) + el-header(60px) + el-main；侧栏标题「第三方核查机构」，主题色 #1e3a5f；头部面包屑 + 头像 + **返回门户**（统一指向 `pcf.html` 产品碳足迹应用门户，与供应商端一致）。

## 四、与运营端/供应商端的关系

- 待核查任务与运营端任务管理中的「核查」阶段（VERIFY）对应；核查机构端任务列表中的 **taskId/orderNo** 与运营端同一任务一致，任务详情与运营端 task_detail_verify 协作日志、报告管理可互相追溯。核查机构完成通过/驳回/澄清后，运营端任务状态与报告管理衔接（参见 01、04、08 文档）。
- **供应商证照**：核查任务详情页提供「供应商证照（仅供核查）」区块，展示当前任务关联供应商的企业名称、统一社会信用代码（可选）、营业执照（副本）等证照的查看/预览；数据来源为**可信数据空间或运营端已同步数据**（与中心端供应商详情的「TDS Synced」一致），按 taskId → 供应商 → 证照 拉取，核查端不维护供应商主数据。
- Mock 数据与 [07_Mock数据说明.md](07_Mock数据说明.md) 及运营端 MOCK_TASK_MAP、订单/任务列表保持一致；**推荐演示路径**见 07 文档「7.2 推荐演示路径」（核查阶段 ↔ 认证端），演示时使用同一批 taskId 可走通闭环。
