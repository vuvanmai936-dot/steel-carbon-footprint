# 线下样表对接说明

本文档说明线上模板/采集表与线下参考样表的对应关系，便于供应商对照填写与运营配置时保持结构一致。

## 1. 参考样表

| 文件名 | 用途 |
|--------|------|
| `0815-石墨电极-产品数据收集表—拆工序小规格.xlsx` | 石墨电极产品、按工序拆解的数据收集表（小规格） |
| `20270724---数据收集模板.xlsx` | 通用数据收集模板 |

## 2. 线上统一结构（与 docs/05 一致）

- **Sheet 与工序**：每个 Sheet 对应一个工序，Sheet 名称即工序名（如 M配料、M焙烧、M石墨化）。
- **表头（6 列）**：`类别`、`名称`、`单位`、`填报指引`、`数值`、`备注`。
- **列索引**：0 类别、1 名称、2 单位、3 填报指引、4 数值、5 备注；供应商可编辑列为 4、5（数值、备注）。

## 3. 映射与校验约定

- 上传 Excel 时（模板管理「Excel 导入」或模板详情「导入 Excel」），系统按 **Sheet 名称 → 工序**、**首行 → 表头** 解析。
- 若线下样表列名与上述 6 列不完全一致，建议：
  - 在导入前将样表首行调整为：类别、名称、单位、填报指引、数值、备注（缺列可留空或补列）；
  - 或在后端/前端解析层做列名映射（如「填报说明」→「填报指引」），将解析结果归一为 6 列后再写入 `sheetData`。
- 工序名（Sheet 名）建议与线下样表保持一致（如 M配料、M焙烧），便于供应商按线下表逐工序填写。

## 4. 与解析逻辑的衔接

- Excel 解析逻辑见 [05_模板引擎解析逻辑](05_模板引擎解析逻辑.md)：Sheet → 工序，首行 → 字段名。
- 解析得到的 `sheetData` 为 `{ "工序名": [ [表头行], [数据行], ... ] }`，与 Snapshot 格式一致，可直接用于 SpreadJS 展示与任务下发。

## 5. 供应商填报页 Excel 导入说明

- 供应商在「任务填报」页可**下载填报模板**（与当前任务下发的 Snapshot 结构一致），在本地 Excel 中填写**数值**、**备注**列后，通过**上传已填 Excel** 回填到页面。
- **要求**：上传的 Excel 的 Sheet 名称须与模板完全一致（如 M配料、M焙烧）；每个 Sheet 的首行须为上述 6 列：类别、名称、单位、填报指引、数值、备注。系统仅回填第 5、6 列（数值、备注），不会覆盖模板下发的类别、名称、单位、填报指引。
- 凭证仍在填报页「凭证上传」区单独上传，不可通过 Excel 一并上传。

## 6. 后续扩展

- 若需严格校验线下样表与线上结构一致性，可在解析后增加校验步骤：检查 Sheet 数量、首行列名与标准 6 列对应关系，不符合时提示并给出调整建议。
