# 接口与互操作设计（实施期）

本文档为**正式实施阶段**的技术方案占位，用于与 NDI《数据基础设施 接入连接器技术要求》及互联互通基本要求对齐；原型阶段可不实现，但需在设计中预留。与 [07_Mock数据说明](07_Mock数据说明.md) 的区别在于：07 描述应用内 Mock 与联调约定，本文档描述与 TDS/NDI 的**对外接口与互操作**。

**第一阶段（MVP）说明**：排期要求本阶段完成平台级「中心连接器」**基础版**开发（轻量框架，具备服务注册与代理能力雏形），**仅作技术储备，此阶段不用于生产**；正式用于委托业务（擎工 SaaS/轻量化）在第二、三阶段。详见 [00_迭代计划/迭代计划与排期说明](../00_迭代计划/迭代计划与排期说明.md)。

---

## 1. 适用范围与依据

- **依据**：NDI《数据基础设施 接入连接器技术要求》《数据基础设施 互联互通基本要求》。
- **范围**：本系统作为业务应用，需与行业功能节点（服务平台）及各方接入连接器对接的接口范围；接口协议与数据格式需满足「统一接口要求」。

---

## 2. 对接对象

- **行业功能节点（服务平台）**：身份、目录、数字合约、使用控制策略、审计上报等能力。
- **供应商/认证机构侧接入连接器**：任务与报告数据、证照同步、消息等。
- **可选**：与南钢 LCA、商城等外部系统的接口（可与现有 [04_任务调度与状态机](../01_业务与流程/04_任务调度与状态机.md) 中 2.5 节 LCA 接口描述合并引用）。

---

## 3. 接口与数据格式原则

- **协议**：如 HTTPS、约定 REST/消息格式。
- **认证方式**：与身份管理对接（统一身份登记结果、DID 等）。
- **数据模型与标识**：与统一目录标识、数据目录描述规范一致。
- **异构连接器互操作**：需支持的最小能力集包括——注册、心跳、数据推送/拉取、策略下发、日志上报等。

---

## 4. 实施期工作

正式开发时据此细化接口定义（如 OpenAPI）、数据字典与示例，并与连接器/平台提供方联调；原型阶段仅保留本文档作为占位与索引。

---

## 5. SaaS 接入接口（第二阶段预留）

委托业务接入—擎工互联 SaaS 化服务时，平台与擎工 SaaS 的接口范围与负责方如下（待接口文档定稿后替换为正式 API 说明）：

- **任务下发**：平台经中心连接器向擎工 SaaS 下发任务参数（订单/任务标识、产品信息、采集数据或快照引用等）；负责方：荣泽科技（平台/连接器）、擎工互联（SaaS 接收与解析）。
- **状态回调**：擎工 SaaS 向平台回传任务执行状态（如已接收、计算中、已完成、异常）；负责方：擎工互联提供回调接口规范，荣泽科技实现连接器接收与任务状态同步。
- **结果回传**：核算/核查结果、报告或凭证回传至平台，进入报告管理流程；负责方：擎工互联（结果生成与推送）、荣泽科技（接收、落库与报告管理衔接）。

接口细节（请求/响应格式、鉴权、重试与幂等）由双方在第二阶段技术方案与接口文档定稿时确认；本文档仅作占位与职责划分。

---

## 6. 轻量化部署与混合流程（第三阶段预留）

**模式**：供应商本地计算 + 中心端任务调度 + 第三方本地核查。边端部署轻量程序，负责本地任务触发、数据打包与上传；中心连接器扩展后支持边端接入连接器的注册、管理与通信；平台侧混合流程编排引擎协调「供应商本地计算」与「中心调度」与「第三方本地核查」的混合流程。

- **边端-云端数据流转协议**：数据内容、格式、加密与签名机制由三方（鑫智链/荣泽/擎工）在第三阶段方案评审时定稿。
- **职责归属**：轻量化部署版本程序与边端 LCA 计算库/镜像由擎工互联提供；中心连接器能力扩展、混合流程编排引擎由荣泽科技（碳足迹建设团队/基础设施团队）负责。详细技术方案见第三阶段评审文档。
