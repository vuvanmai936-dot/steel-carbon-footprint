# 接口与互操作设计（实施期）

本文档为**正式实施阶段**的技术方案占位，用于与 NDI《数据基础设施 接入连接器技术要求》及互联互通基本要求对齐；原型阶段可不实现，但需在设计中预留。与 [07_Mock数据说明](07_Mock数据说明.md) 的区别在于：07 描述应用内 Mock 与联调约定，本文档描述与 TDS/NDI 的**对外接口与互操作**。

> **本期更新（2026-05-09）**：新增 §7「最小 OpenAPI 片段」作为本系统**应用内核心接口契约 SoT**，给六仓 fe/back 用作字段对齐的入场券。如有不一致，以本文为准并按 [`SoT_MAP`](../SoT_MAP.md) 流程更新。

**第一阶段（MVP）说明**：排期要求本阶段完成平台级「中心连接器」**基础版**开发（轻量框架，具备服务注册与代理能力雏形），**仅作技术储备，此阶段不用于生产**；正式用于委托业务（擎工 SaaS/轻量化）在第二、三阶段。详见 [00_迭代计划/迭代计划与排期说明](../00_迭代计划/迭代计划与排期说明.md)。

---

## 1. 适用范围与依据

- **依据**：NDI《数据基础设施 接入连接器技术要求》《数据基础设施 互联互通基本要求》。
- **范围**：本系统作为业务应用，需与行业功能节点（服务平台）及各方接入连接器对接的接口范围；接口协议与数据格式需满足「统一接口要求」。

---

## 2. 对接对象

- **行业功能节点（服务平台）**：身份、目录、数字合约、使用控制策略、审计上报等能力。
- **供应商/认证机构侧接入连接器**：任务与报告数据、证照同步、消息等。
- **可选**：与南钢 LCA、商城等外部系统的接口（可与现有 [04_任务调度与状态机](../01_业务与流程/04_任务调度与状态机.md) 中 2.5 节 LCA 接口描述合并引用）。

---

## 3. 接口与数据格式原则

- **协议**：如 HTTPS、约定 REST/消息格式。
- **认证方式**：与身份管理对接（统一身份登记结果、DID 等）。
- **数据模型与标识**：与统一目录标识、数据目录描述规范一致。
- **异构连接器互操作**：需支持的最小能力集包括——注册、心跳、数据推送/拉取、策略下发、日志上报等。

---

## 4. 实施期工作

正式开发时据此细化接口定义（如 OpenAPI）、数据字典与示例，并与连接器/平台提供方联调；原型阶段仅保留本文档作为占位与索引。

## 5. SaaS 接入接口（第二阶段预留）

委托业务接入—擎工互联 SaaS 化服务时，平台与擎工 SaaS 的接口范围与负责方如下（待接口文档定稿后替换为正式 API 说明）：

- **任务下发**：平台经中心连接器向擎工 SaaS 下发任务参数（订单/任务标识、产品信息、采集数据或快照引用等）；负责方：荣泽科技（平台/连接器）、擎工互联（SaaS 接收与解析）。
- **状态回调**：擎工 SaaS 向平台回传任务执行状态（如已接收、计算中、已完成、异常）；负责方：擎工互联提供回调接口规范，荣泽科技实现连接器接收与任务状态同步。
- **结果回传**：核算/核查结果、报告或凭证回传至平台，进入报告管理流程；负责方：擎工互联（结果生成与推送）、荣泽科技（接收、落库与报告管理衔接）。

接口细节（请求/响应格式、鉴权、重试与幂等）由双方在第二阶段技术方案与接口文档定稿时确认；本文档仅作占位与职责划分。

---

## 6. 轻量化部署与混合流程（第三阶段预留）

**模式**：供应商本地计算 + 中心端任务调度 + 第三方本地核查。边端部署轻量程序，负责本地任务触发、数据打包与上传；中心连接器扩展后支持边端接入连接器的注册、管理与通信；平台侧混合流程编排引擎协调「供应商本地计算」与「中心调度」与「第三方本地核查」的混合流程。

- **边端-云端数据流转协议**：数据内容、格式、加密与签名机制由三方（鑫智链/荣泽/擎工）在第三阶段方案评审时定稿。
- **职责归属**：轻量化部署版本程序与边端 LCA 计算库/镜像由擎工互联提供；中心连接器能力扩展、混合流程编排引擎由荣泽科技（碳足迹建设团队/基础设施团队）负责。详细技术方案见第三阶段评审文档。

## 7. 最小 OpenAPI 片段（应用内核心接口契约 SoT）

> **范围**：本节是**应用内**核心业务接口的最小契约，覆盖任务五段（含报告子状态）与澄清/驳回回路；与 §1–§6 描述的 TDS/NDI 对外接口正交。  
> **状态**：v0.1 草案；字段命名与状态枚举锁定 [`docs/01_业务与流程/02_全局数据字典与枚举.md`](../01_业务与流程/02_全局数据字典与枚举.md)、[`docs/01_业务与流程/04_任务调度与状态机.md`](../01_业务与流程/04_任务调度与状态机.md)。  
> **如何使用**：六仓 fe/back 在新增/修改字段前先在此对齐；落地后回填实际 path/method/auth 与差异，并在 [`docs/00_迭代计划/设计变更记录.md`](../00_迭代计划/设计变更记录.md) 登记一行。

### 7.1 端点一览

| # | Method & Path | 用途 | 主要消费方（页面） | 对应开发仓 |
|---|----------------|------|---------------------|------------|
| 1 | `GET /api/tasks` | 任务列表（自营/委托，按角色过滤） | `operator/self_operated_task_list.html`、`supplier/task_list.html`、`certifier/task_list.html` | `pcf-center-back` |
| 2 | `GET /api/tasks/{taskId}` | 任务详情（含 stageIndex 与子状态） | 三端 `task_detail*` | `pcf-center-back` |
| 3 | `POST /api/tasks/{taskId}/collect:submit` | 供应商提交采集（含 snapshot 引用） | `supplier/task_fill.html` | `pcf-supplier-back` |
| 4 | `POST /api/tasks/{taskId}/collect:audit` | 运营/链主对采集做通过/驳回 | `operator/task_detail_collect.html` | `pcf-center-back` |
| 5 | `POST /api/tasks/{taskId}/verify:decide` | 核查机构对核查节点做通过/驳回 | `certifier/task_detail_verify.html` | `pcf-audit-back` |
| 6 | `POST /api/reports/{reportId}/issue` | 运营对报告执行下发 | `operator/report_mgt.html` | `pcf-center-back` |
| 7 | `POST /api/reports/{reportId}/confirm` | 供应商确认接收报告 | `supplier/reports.html` | `pcf-supplier-back` |
| 8 | `POST /api/reports/{reportId}/appeal` | 供应商对报告发起申诉（仅"已下发未确认"窗口） | `supplier/reports.html` | `pcf-supplier-back` + `pcf-center-back` |

### 7.2 OpenAPI 3.1 片段

```yaml
openapi: 3.1.0
info:
  title: PCF Internal API (Minimal Contract)
  version: 0.1.0
  description: |
    钢铁行业产品碳足迹平台「应用内」核心契约。覆盖任务五段与报告子状态；
    与 TDS/NDI 互联互通接口正交，正交接口在本文 §1–§6 描述。
servers:
  - url: https://pcf.example/internal
    description: 占位；六仓部署时按内网域名替换

components:
  securitySchemes:
    bearerAuth:
      type: http
      scheme: bearer
      bearerFormat: JWT

  schemas:
    TaskStageIndex:
      type: integer
      enum: [0, 1, 2, 3, 4]
      description: |
        0=配置 1=采集（含审核） 2=计算（LCA） 3=核查 4=报告
        详见 docs/01_业务与流程/04_任务调度与状态机.md §1
    ReportSubStatus:
      type: string
      enum: [pending, issued, pending_archive, archived, void]
      description: |
        阶段 4 子状态：待处理/已下发/待归档/已归档/已作废；语义见上述同一文档 §2.6
    AuditDecision:
      type: string
      enum: [approve, reject]
    Task:
      type: object
      required: [taskId, orderNo, stageIndex]
      properties:
        taskId: { type: string, example: TSK-2026-888 }
        orderNo: { type: string, example: ORD-20260203-001 }
        productName: { type: string }
        supplierName: { type: string }
        certifierName: { type: string, nullable: true }
        stageIndex: { $ref: '#/components/schemas/TaskStageIndex' }
        reportSubStatus:
          $ref: '#/components/schemas/ReportSubStatus'
          description: 仅当 stageIndex=4 时返回
        slaDueAt: { type: string, format: date-time, nullable: true }
        updatedAt: { type: string, format: date-time }
    SnapshotRef:
      type: object
      required: [snapshotId, version]
      properties:
        snapshotId: { type: string, example: SNAP-2026-0001 }
        version: { type: integer, example: 3 }
        templateCode: { type: string, example: TPL-STEEL-V1 }
    Report:
      type: object
      required: [reportId, taskNo, status]
      properties:
        reportId: { type: string, example: RPT-2026-0001 }
        taskNo: { type: string, description: 与 Task.taskId 一一对应 }
        title: { type: string }
        status: { $ref: '#/components/schemas/ReportSubStatus' }
        issuedAt: { type: string, format: date-time, nullable: true }
        confirmedAt: { type: string, format: date-time, nullable: true }
    Error:
      type: object
      required: [code, message]
      properties:
        code: { type: string, example: TASK_STATE_INVALID }
        message: { type: string }
        details: { type: object, additionalProperties: true }

security:
  - bearerAuth: []

paths:
  /api/tasks:
    get:
      summary: 任务列表
      parameters:
        - in: query
          name: role
          required: true
          schema: { type: string, enum: [operator, supplier, certifier] }
        - in: query
          name: stageIndex
          schema: { $ref: '#/components/schemas/TaskStageIndex' }
        - in: query
          name: keyword
          schema: { type: string }
        - in: query
          name: page
          schema: { type: integer, default: 1 }
        - in: query
          name: pageSize
          schema: { type: integer, default: 20 }
      responses:
        '200':
          description: ok
          content:
            application/json:
              schema:
                type: object
                properties:
                  total: { type: integer }
                  items:
                    type: array
                    items: { $ref: '#/components/schemas/Task' }

  /api/tasks/{taskId}:
    get:
      summary: 任务详情
      parameters:
        - in: path
          name: taskId
          required: true
          schema: { type: string }
      responses:
        '200':
          description: ok
          content:
            application/json:
              schema: { $ref: '#/components/schemas/Task' }
        '404':
          description: 任务不存在
          content:
            application/json:
              schema: { $ref: '#/components/schemas/Error' }

  /api/tasks/{taskId}/collect:submit:
    post:
      summary: 供应商提交采集
      parameters:
        - in: path
          name: taskId
          required: true
          schema: { type: string }
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              required: [snapshot]
              properties:
                snapshot: { $ref: '#/components/schemas/SnapshotRef' }
                attachments:
                  type: array
                  items: { type: string, description: 附件 ID 或对象存储 URL }
                comment: { type: string }
      responses:
        '200': { description: ok }
        '409':
          description: 阶段不允许提交（需在 stageIndex=1 且未提交）
          content:
            application/json:
              schema: { $ref: '#/components/schemas/Error' }

  /api/tasks/{taskId}/collect:audit:
    post:
      summary: 采集审核（运营/链主）
      parameters:
        - in: path
          name: taskId
          required: true
          schema: { type: string }
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              required: [decision]
              properties:
                decision: { $ref: '#/components/schemas/AuditDecision' }
                rejectReason: { type: string, description: decision=reject 时必填 }
                clarificationId: { type: string, nullable: true }
      responses:
        '200': { description: ok }

  /api/tasks/{taskId}/verify:decide:
    post:
      summary: 核查决策（核查机构）
      parameters:
        - in: path
          name: taskId
          required: true
          schema: { type: string }
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              required: [decision]
              properties:
                decision: { $ref: '#/components/schemas/AuditDecision' }
                rejectReason: { type: string }
      responses:
        '200': { description: ok }

  /api/reports/{reportId}/issue:
    post:
      summary: 报告下发（运营）
      parameters:
        - in: path
          name: reportId
          required: true
          schema: { type: string }
      responses:
        '200':
          description: ok
          content:
            application/json:
              schema: { $ref: '#/components/schemas/Report' }

  /api/reports/{reportId}/confirm:
    post:
      summary: 供应商确认接收
      parameters:
        - in: path
          name: reportId
          required: true
          schema: { type: string }
      responses:
        '200':
          description: ok
          content:
            application/json:
              schema: { $ref: '#/components/schemas/Report' }
        '409':
          description: 仅"已下发未确认"窗口可调用
          content:
            application/json:
              schema: { $ref: '#/components/schemas/Error' }

  /api/reports/{reportId}/appeal:
    post:
      summary: 供应商申诉
      parameters:
        - in: path
          name: reportId
          required: true
          schema: { type: string }
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              required: [reason]
              properties:
                reason: { type: string, minLength: 1 }
                attachments:
                  type: array
                  items: { type: string }
      responses:
        '200': { description: ok }
        '409':
          description: 仅"已下发未确认"窗口可调用；其余阶段返回该错误
          content:
            application/json:
              schema: { $ref: '#/components/schemas/Error' }
```
