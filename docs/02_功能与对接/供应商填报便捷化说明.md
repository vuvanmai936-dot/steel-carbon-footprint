# 供应商填报便捷化说明

本文档记录供应商端数据采集便捷化能力的入口、规则与 Mock 接口，便于后续对接真实 API 时替换。

## 1. Excel 导入/导出

- **入口**：任务填报页 [supplier/task_fill.html](supplier/task_fill.html) 工序数据区工具栏。
- **下载填报模板**：按当前任务 Snapshot 的 `sheetData` 生成 6 列标准表头 Excel，Sheet 名与工序一致；文件名 `填报模板_{taskId}_{productName}.xlsx`。
- **上传已填 Excel**：解析后校验 Sheet 名与首行（类别、名称、单位、填报指引、数值、备注）与模板一致，仅回填数值、备注列；凭证仍在页内单独上传。
- **依赖**：SheetJS (XLSX)、[spreadUtils.js](../js/spreadUtils.js) 的 `parseExcelFile`、`exportSnapshotToExcelBlob`、`validateUploadedSheetData`、`applyUploadedSheetDataToSpread`。表头与结构约定见 [10_线下样表对接说明](10_线下样表对接说明.md)。

## 2. 历史数据与默认值复用

- **入口**：进入任务填报页时若存在该任务历史提交数据，弹窗提示「是否带出数值与备注」。
- **逻辑**：使用 [mockTasks.js](../js/mockTasks.js) 的 `getTaskSubmittedSnapshot(taskId)`；若返回非空则提示，用户选「带出」后调用 `applyUploadedSheetDataToSpread` 写入可编辑列。
- **Mock**：`getTaskSubmittedSnapshot`（已有）；提交后写入 `MOCK_TASK_SUBMITTED`，与采集审核共用。

## 3. 多任务合并填报

- **入口**：我的订单 → 订单详情；当该订单下存在不少于 2 个可合并填报任务时展示「合并填报」按钮，跳转 [supplier/task_fill_batch.html](supplier/task_fill_batch.html)?orderNo=xxx。
- **可合并任务**：同一订单、同一供应商、阶段为采集(stageIndex===1)、待填报或采集已驳回、且已下发模板的任务。
- **页内**：按任务 Tab 展示，每个 Tab 内为工序表（SpreadJS）+ 该任务凭证上传；切换 Tab 时自动保存当前表格到对应任务草稿；底部「提交全部」逐条调用 `submitTaskData`。
- **Mock**：`getTasksByOrderNoForSupplier(orderNo, supplierName)` 返回上述任务列表及每任务 `snapshot`；无新提交接口，仍用现有 `submitTaskData` 逐条提交。

## 4. 凭证按需上传与草稿保存

- **入口**：任务填报页「保存草稿」按钮；再次进入该任务时若存在草稿则提示「是否恢复草稿」。
- **草稿内容**：当前工序表 Snapshot + 凭证列表引用；恢复时仅恢复表格数据，凭证需重新上传。
- **提交**：提交成功时清除该任务草稿；草稿不校验凭证必填。
- **Mock**：`saveTaskDraft(taskId, snapshot, evidenceFiles)`、`getTaskDraft(taskId)`；存储于 `MOCK_TASK_DRAFT`；`submitTaskData` 内会删除对应 taskId 草稿。

## 5. 接口与后续对接

| 能力       | Mock 函数 / 存储              | 后续可替换为                         |
|------------|-------------------------------|----------------------------------------|
| Excel 导出 | SpreadUtils.exportSnapshotToExcelBlob | 后端生成模板下载链接或流              |
| Excel 导入 | SpreadUtils.parseExcelFile + 校验 + apply | 后端解析上传文件返回 sheetData 或前端保持 |
| 历史带出   | getTaskSubmittedSnapshot      | GET /api/tasks/:id/last-submission     |
| 合并列表   | getTasksByOrderNoForSupplier  | GET /api/orders/:orderNo/fillable-tasks |
| 草稿       | saveTaskDraft / getTaskDraft  | POST/GET 草稿接口                      |
