## 一、快速开始

### 1.1 系统是什么

本系统为**钢铁行业产业链可信数据空间产品碳足迹数据服务系统**的供应商端，即「供应商工作台」。您在此接收任务、填报数据与凭证、处理澄清或驳回、确认报告或提起异议。

### 1.2 您在本系统中的角色

您作为 **L2 供应商（数源方）**，核心工作包括：填报数据、上传凭证、配合澄清/整改、确认报告或提起异议。

### 1.3 典型操作路径（三步）

1. 在「我的任务」中处理**待填报**或**待澄清/待整改**任务，进入填报页或澄清页完成数据填写、凭证上传或回复说明。
2. 在「我的碳报告」中查看运营已下发的报告，预览后选择**确认接收**或**我有异议**（申诉）。
3. 在「企业信息」中维护企业基本信息与证照（营业执照必填），并**提交至可信数据空间**，供运营与核查方使用。

### 1.4 入口与返回

侧栏可切换工作台、我的任务、我的订单、我的碳报告、企业信息等；顶部「返回门户」可返回产品碳足迹应用门户。

## 二、业务与背景

### 2.1 产品碳足迹 (PCF) 与 LCA

- **产品碳足迹（PCF）**：产品在其整个生命周期中所产生的温室气体排放的量化与报告。
- **LCA（生命周期评价）**：本系统中由运营侧/南钢 LCA 系统完成计算。
- **供应商职责**：您提供的是**上游或工序级原始数据与凭证**，用于核算产品碳足迹。

### 2.2 可信数据空间 (TDS) 与本系统

- 本系统运行在**钢铁行业产业链可信数据空间**上。
- 数据在空间内流通，受数字合约与使用控制约束。
- 您的**企业信息与证照**需「提交至空间」，供运营端与核查机构使用；填报与查看行为受「使用可信」约束，仅限规定范围内使用。

### 2.3 任务从哪来

订单由商城或运营侧产生；运营确认并分配任务后，您会在「我的任务」中看到需要处理的任务，类型包括：**待填报**、**采集已驳回**、**待澄清/待整改**。

## 三、功能使用指南

| 模块 | 说明要点 |
|------|----------|
| **工作台** | 待办数量、待澄清数量、报告数等统计；快捷入口：我的任务、我的碳报告、返回门户。 |
| **我的任务** | 任务类型：待填报、采集已驳回、待澄清/待整改。列表展示任务编号、订单号、产品、规格、类型/状态、最新通知；点击进入「填报」或「澄清」页。 |
| **任务填报** | 产品与订单为只读；按工序填报数据、上传凭证。若为「采集已驳回」请先查看驳回原因与整改要求，修改后重新提交。提交后由运营审核，通过后进入 LCA 计算。 |
| **待澄清处理** | 任务与产品只读；澄清在采集、计算、核查等多阶段都可能出现，运营或核查方转派给您后您在此回复说明并上传佐证。若同时被「采集驳回」，任务会进入「整改中」，需在填报页重填并提交。 |
| **我的碳报告** | 仅展示**运营已下发**的报告。操作：预览 → 二选一「确认接收」或「我有异议」。确认接收后**不可再申诉**；申诉由运营在报告管理中驳回或接受。 |
| **我的订单** | 订单列表与状态查看；与任务关联的订单号可在任务详情/填报页查看。 |
| **企业信息与证照** | 维护企业基本信息、联系人、证照（营业执照必选，ISO 等可选）。「保存」「提交至空间」将数据写入可信数据空间；支持「从空间同步」查看（若已采用空间同步方式）。 |

## 四、流程与状态

### 4.1 任务五阶段

配置（运营）→ 采集（您填报与运营审核）→ 计算（LCA）→ 核查（第三方）→ 报告（下发、确认、归档）。您主要参与「采集」与「报告」阶段；**澄清**在采集、计算、核查等多阶段都可能出现（运营或核查方有疑问时会要求您说明或补充）；**驳回**您会直接遇到的主要是「采集被驳回」与「报告申诉被驳回」。

### 4.2 澄清与驳回（多阶段）

- **澄清**：在**采集、计算（LCA）、核查**等阶段都可能出现。运营或核查方对数据/凭证有疑问时，会通过澄清要求您说明或补充佐证；若被转派给您，您会在「待澄清/待整改」中看到并回复。澄清仅需回复说明，一般不改变阶段状态。
- **驳回**：**采集阶段**—运营对您提交的数据审核不通过时，任务变为「整改中」，需按驳回原因修改数据或凭证后重新提交；**报告阶段**—您对报告提起申诉后，运营可能「驳回申诉」（维持原报告）。核查阶段的验收驳回责任方为核查机构，您无需直接操作。

### 4.3 采集阶段子状态

待提交、审核中、整改中、已完成。

### 4.4 报告阶段

运营下发后您可在「我的碳报告」中看到该报告 → 预览 → 选择「确认接收」（之后不可申诉）或「我有异议」（申诉）。待归档/已归档仅在您确认接收后由运营操作，您无需在报告页做其他动作。

## 五、安全与合规

### 5.1 数据使用控制

填报与查看在系统内完成，受「使用可信」约束：**仅限在线查看与填报**，禁止下载、截图、导出等。正式环境由策略引擎/连接器执行上述约束。

### 5.2 企业信息与证照

请确保企业信息与证照**真实有效、必填项齐全**。提交至空间后供运营与核查机构使用，请勿造假。

## 六、常见问题 FAQ

| 问题 | 答案要点 |
|------|----------|
| 什么是 PCF？什么是 LCA？我在其中提供什么？ | PCF 是产品全生命周期温室气体排放的量化与报告；LCA 由运营侧/LCA 系统完成计算。您作为供应商提供工序级原始数据与凭证。 |
| 什么是可信数据空间？为什么企业信息要「提交至空间」？ | 可信数据空间是支撑数据安全流通的基础设施。企业信息与证照提交至空间后供运营与核查机构使用，满足身份与数据合规要求。 |
| 待办任务里「待填报」「采集已驳回」「待澄清」分别是什么意思？ | **待填报**：需填写工序数据并上传凭证后提交。**采集已驳回**：运营审核未通过，需按驳回原因修改后重新提交。**待澄清/待整改**：需在澄清页回复说明或补充佐证。 |
| 如何填报任务数据？驳回后该怎么办？ | 在「我的任务」中进入填报页，按工序填写并上传凭证。若为驳回任务，先查看驳回原因与整改要求，修改后重新提交。 |
| 澄清和驳回有什么区别？我只收到澄清时要不要改数？ | **澄清**：在采集、计算（LCA）、核查等多阶段都可能出现，仅需在澄清页回复说明或补充佐证，一般不要求改数。**驳回**：采集阶段驳回=运营审核不通过，需改数或换凭证后重新提交；报告阶段驳回=您申诉后运营驳回申诉（维持原报告）。若仅收到澄清且未被驳回，按澄清要求回复即可，通常无需改数。 |
| 报告什么时候能看到？确认接收和申诉只能二选一吗？确认后还能申诉吗？ | 报告仅在**运营下发后**出现在「我的碳报告」中。预览后只能二选一：确认接收或我有异议。一旦「确认接收」则**不可再申诉**。 |
| 企业信息里「提交至空间」失败或同步不到怎么办？ | 请检查网络与必填项（如营业执照）。若仍无法提交或同步，请联系运营管理员或见门户公示的支持联系方式。 |
| 填报页提示「仅限在线查看与填报」「禁止下载或截图」是什么意思？ | 数据使用受「使用可信」约束：只能在系统内在线查看与填报，不能下载、截图或导出，以保障数据安全与合规。 |

## 后续功能调整时的更新方式

当功能或流程有调整时，可按以下方式保持知识库与帮助内容一致：

1. **人工同步**：依据本文档结构与 06、13、18 等业务文档，在本文档中修订对应章节与 FAQ，再同步修改 [supplier/help.html](../../supplier/help.html) 中对应区块与折叠项。
2. **由脚本自动同步**：修改本文档后，在项目根目录执行 **`npm run sync-help`**（或 `node scripts/sync-help-from-doc.js`），脚本会读取本文档并自动更新 [supplier/help.html](../../supplier/help.html) 的正文区块（快速开始、业务与背景、功能指南、流程与状态、安全与合规、FAQ、联系与支持），无需手工改 HTML。脚本见 [scripts/sync-help-from-doc.js](../../scripts/sync-help-from-doc.js)。
3. **由 AI/开发编写**：若功能或流程有较大调整，可先更新 06、13、18 等业务文档，再委托开发或 AI 按本文档结构更新本文档，然后执行 `npm run sync-help` 同步到 help.html。
