# 任务详情页统一实施方案（供审核）

本文档为**完整可执行方案**，用于从「规则配置」页开始，统一各任务详情页的头部与信息条，供产品/开发审核后按项实施。

---

## 一、目标与范围

### 1.1 目标

- **头部（el-header）**：各任务详情页左侧（返回 + 面包屑）、右侧（SLA + 头像 + 用户名）结构一致，样式统一。
- **任务信息条（task-info-dashboard）**：前两块「产品与任务」「来源供应商」的 HTML 结构、用词、交互（订单链接、供应商链接）全站一致；第三块「XX 状态」标题规范统一；第四块按业务保留差异。
- **辅助**：确认各页引用 `task_detail.css`，goBack / viewOrder / viewSupplier 行为一致。

### 1.2 涉及页面

| 序号 | 页面 | 文件 | 角色 |
|------|------|------|------|
| 1 | 规则配置 | operator/task_detail_config.html | **基准页**（先按规范微调，再作模板） |
| 2 | 数据采集 | operator/task_detail_collect.html | 对齐基准 |
| 3 | LCA 计算 | operator/task_detail_lca.html | 对齐基准 |
| 4 | 核查认证 | operator/task_detail_verify.html | 对齐基准 |
| 5 | 报告引导 | operator/task_detail_certify.html | 仅头部与基准一致（可选 SLA 占位），现已有统一信息条 |
| 6 | 核查端任务详情 | certifier/task_detail.html | 三块 + 操作区 + TDS 说明 |
| 7 | 供应商填报 | supplier/task_fill.html | 三块 + 提交操作 + TDS 说明 |

### 1.3 任务信息条三块布局与 TDS 说明（已落地）

- **Block 1（任务与上下文）**：产品名称、任务 ID、规格、订单（链接）、来源供应商（链接）；可选 DID/企业信息。聚合企业、产品、订单、任务等多类信息。
- **Block 2（状态与待办）**：当前阶段状态 Tag、SLA 剩余时间、消息与澄清入口；待办事项或驳回原因按业务展示。
- **Block 3（操作）**：按端与阶段展示主操作（如 通过/驳回、挂起/驳回/核验通过&签发、前往报告管理、提交、待办说明等）。
- **可信数据空间（TDS）说明条**：在任务信息条下方固定展示一条说明，强调数据安全与隐私：「可信数据空间：本任务数据仅限授权范围内使用，禁止未授权下载、截图与传播；数据安全与隐私受数字合约保护。」样式类名为 `task-info-dashboard__tds`，见 `css/task_detail.css`。

## 二、基准规范（以 config 为准）

### 2.1 头部（el-header）标准结构

**布局**：`display: flex; justify-content: space-between; align-items: center;`，高度 60px，背景 #fff，下边框 1px #dcdfe6，左右 padding 20px。

**左侧区域**（顺序固定）：

1. **返回按钮**：`<el-button icon="ArrowLeft" circle @click="goBack">`，margin-right: 15px。
2. **面包屑**：`<el-breadcrumb separator="/">`
   - 第 1 项：`中心运营端`
   - 第 2 项：`任务管理`
   - 第 3 项：**当前阶段名**（见下表，各页不同）

| 页面 | 面包屑第 3 项 |
|------|----------------|
| 规则配置 | 配置规则 |
| 数据采集 | 数据采集 |
| LCA 计算 | LCA 计算 |
| 核查认证 | 核查验收 |
| 报告引导 | 报告（已统一至报告管理） |

**右侧区域**（顺序固定）：

1. **SLA 胶囊**（certify 页可为占位，见 2.2）  
   - 结构：`<el-tooltip content="SLA 剩余时间" placement="bottom"><div class="sla-capsule"><el-icon><Timer /></el-icon> HH:MM:SS</div></el-tooltip>`  
   - 文案：统一为「SLA 剩余时间」（不强制带截止日期，避免各页不一致）；数值可暂时写死，后续接 Mock/接口。
2. **头像**：`<el-avatar :size="32" style="background:#409EFF; margin-right:8px">运营</el-avatar>`
3. **用户名**：`<span style="font-size: 14px; color: #606266;">{{ currentUser.name }}</span>`

**certify 页**：若保持「引导页」定位，右侧可**不展示 SLA**（仅头像+用户名）；若需与其他详情页完全一致，可增加灰色占位文案「—」或「不适用」，不展示倒计时。

### 2.2 任务信息条（task-info-dashboard）标准结构

**容器**：`class="task-info-dashboard"`，使用 `css/task_detail.css` 已有样式。内部为横向若干 `dash-block`，建议 4 块。

---

#### 第一块：产品与任务（所有有信息条的页统一）

- **标题**：`<div class="dash-title">产品与任务</div>`
- **内容**：
  - **prod-main**：`<span class="prod-name">{{ taskInfo.productName }}</span>` + `<span class="prod-id" style="...">{{ taskInfo.taskId }}</span>`（prod-id 样式与 config 一致：monospace, 12px, background #f4f4f5, padding 2px 6px, border-radius 4px, margin-left 8px）
  - **prod-sub**（必须包含以下三项，顺序固定）：
    1. `规格: {{ taskInfo.specs }}`（span，margin-right 10px）
    2. 分隔符 `|`（span，color #dcdfe6，margin-right 10px）
    3. 订单：`<el-icon><Tickets /></el-icon>` + `订单: ` + `<el-link type="primary" :underline="false" style="font-size:12px; margin-left:2px" @click="viewOrder">{{ taskInfo.orderNo }}</el-link>`
  - 需确保各页均注册 `viewOrder`（跳转订单页）。

---

#### 第二块：来源供应商（所有有信息条的页统一）

- **标题**：统一为 `<div class="dash-title">来源供应商</div>`（不再使用「目标供应商」）。
- **内容**：
  - 第一行：头像 + 供应商名称。供应商名称**必须**为可点链接：  
    `<el-link type="primary" :underline="false" style="font-weight:bold; font-size:14px;" @click="viewSupplier">{{ taskInfo.supplier }}</el-link>`  
    头像与 config 一致：`<el-avatar :size="24" style="background:#e6f7ff; color:#1890ff; font-size:12px; margin-right:8px;">{{ taskInfo.supplier[0] }}</el-avatar>`
  - 第二行（副文案）：`<span style="background:#f0f9eb; color:#67C23A; padding:0 4px; border-radius:2px; font-size:10px;">Level A</span>` + 空格 + `信用良好`，外层 div 使用 `font-size:12px; color:#909399; margin-top:4px; margin-left:32px;`。
- 需确保各页均注册 `viewSupplier`（跳转供应商管理详情，如 supplier_mgt_detail.html）。

---

#### 第三块：当前阶段状态（标题统一规范，内容各页不同）

- **标题**：必须为「XX 状态」形式，与阶段对应：
  - 规则配置：`配置状态`
  - 数据采集：`采集状态`
  - LCA 计算：`计算状态`（与配置/采集/核查状态命名一致）
  - 核查认证：`核查状态`
- **内容**：各页按业务展示 tag + 说明文案，结构可保留现有实现。

---

#### 第四块：数据/统计（各页保留业务差异）

- **标题**：各页自定（如「规则定义」「数据/凭证」「结果」「文档」等）。
- **内容**：保持现有 stat-item 等结构，不强制统一文案。

## 四、全局约定与检查项

### 4.1 样式与脚本

- 所有上述详情页均引用 `../css/task_detail.css`，保证 `.task-info-dashboard`、`.dash-block`、`.sla-capsule`、`.prod-main`、`.prod-sub` 等生效。
- 若某页未引用，补充：`<link rel="stylesheet" href="../css/task_detail.css">`。

### 4.2 导航与链接

- **goBack**：逻辑统一为：若 URL 带 `from=order` 则跳转 `order.html`，否则跳转 `self_operated_task_list.html`。
- **viewOrder**：跳转 `order.html`（可带 query，与现有一致即可）。
- **viewSupplier**：跳转运营端供应商详情页（如 `supplier_mgt_detail.html`）；config、collect、lca 已有则保持，verify 必须补充。

### 4.3 数据与图标

- 各页 `taskInfo` 来源一致（如 getTaskInfoFromUrl / mockTasks），保证 `productName`、`specs`、`orderNo`、`supplier`、`taskId` 可用。
- 订单图标使用 Element Plus 的 `Tickets`（与 config 一致），确保各页已注册或全局注册该图标。

## 五、实施顺序与验收

### 5.1 建议实施顺序

1. **task_detail_config.html**：按 3.1 修改（SLA 文案、第二块标题「来源供应商」），作为基准。
2. **task_detail_collect.html**：按 3.2 核对/修改。
3. **task_detail_lca.html**：按 3.3 逐项修改（prod-sub、第二块标题与副文案、第三块标题、viewSupplier）。
4. **task_detail_verify.html**：按 3.4 逐项修改（prod-sub、第二块链接与副文案、viewSupplier、goBack）。
5. **task_detail_certify.html**：按 3.5 可选补头部 SLA 占位。

### 5.2 验收标准

- **头部**：任意两个详情页并排对比，左侧（返回+面包屑）、右侧（SLA+头像+用户名）结构一致，仅面包屑第三项和 SLA 数值不同。
- **信息条**：config / collect / lca / verify 四页的前两块（产品与任务、来源供应商）HTML 结构一致，且规格/订单/供应商均为约定文案与可点链接；第三块标题均为「XX 状态」；第四块允许不同。
- **交互**：从各详情页点击订单链接、供应商链接、返回，行为符合 4.2；verify 页点击供应商名称可跳转供应商详情。

---

## 六、审核要点（供产品/开发确认）

1. **供应商块标题**：是否统一为「来源供应商」？（当前 config/lca 为「目标供应商」，collect/verify 为「来源供应商」。）
2. **SLA**：certify 是否需要与其它页保持「右侧三件套」一致并加占位，还是保持仅头像+用户名？
3. **SLA 文案**：是否全部采用「SLA 剩余时间」且不显示截止日期？
4. **第四块**：是否仅约定「标题 + 若干 stat-item」结构，内容各页自定？

以上确认后，即可按第三节「各页具体修改清单」与第五节「实施顺序」执行开发与联调。
