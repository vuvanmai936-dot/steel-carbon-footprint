# 任务阶段与子状态说明

本文档为产品说明，明确任务列表中「当前阶段/进度」及任务详情页顶部「XX 状态」所展示的各阶段、子状态含义及界面文案，便于产品、开发与测试统一理解。

---

## 1. 阶段划分（5 段）

任务采用 **stageIndex 0～4** 表示五个主阶段，与列表进度条、详情页入口一致：

| stageIndex | 阶段名称 | 说明 |
| :---: | :--- | :--- |
| 0 | 配置 | 规则配置与模板快照下发 |
| 1 | 采集 | 数据采集与运营审核 |
| 2 | 计算 | LCA 计算（当前多页详情中块标题为「计算状态」） |
| 3 | 核查 | 三方核查与运营验收 |
| 4 | 报告 | 报告/证书下发、供应商确认、归档上链等，统一在报告管理处理 |

---

## 2. 各阶段子状态（枚举与展示文案）

以下子状态用于列表与详情页**状态展示**，数据来源为统一任务数据（如 `MOCK_TASK_LIST` / 接口），详情页按**当前页面阶段**取对应行，使状态与页面一致。界面文案与 `getStageStatusText` 约定一致。

### 2.1 阶段 0：配置

| 子状态枚举 (stageSubStatus) | 界面展示文案 | 说明 |
| :--- | :--- | :--- |
| `pending` | 待配置模板 | 未初始化或未选模板 |
| `draft` | 配置中 | 已生成快照，未锁定 |
| `locked` | 已配置 | 已下发，配置不可变更 |

### 2.2 阶段 1：采集

| 子状态枚举 (stageSubStatus) | 界面展示文案 | 说明 |
| :--- | :--- | :--- |
| `waiting_submit` | 待提交 | 等待供应商提交数据 |
| `auditing` | 审核中 | 运营审核中 |
| `waiting_update` | 整改中 | 已驳回，供应商整改 |
| `passed` | 已完成 | 审核通过，可进入计算 |

**采集阶段操作边界（提出疑点与驳回）**：运营在审核中可选用两种操作，二者区分如下。

- **提出疑点**：对数据/凭证有疑问、仅需供应商说明时使用。通过澄清渠道发起一条疑点讨论，**不改变采集状态**；供应商在「消息与澄清」中回复即可。
- **驳回**：认为需退回重填时使用。任务状态变为「整改中」，并同步创建「整改要求」类澄清，便于后续沟通与整改说明。

**决策指引**：需要改数或换凭证 → 驳回；只需解释说明 → 提出疑点。

**典型场景示例**：

| 情形 | 建议操作 | 说明 |
| :--- | :--- | :--- |
| 某数值来源不明，需供应商补充说明 | 提出疑点 | 供应商在澄清中回复即可，无需重填表格 |
| 凭证格式不符合要求（如缺盖章），需重新上传 | 驳回 | 供应商需进入填报页重新上传凭证并提交 |
| 对某工序填报口径有疑问，需确认理解是否一致 | 提出疑点 | 澄清回复后若一致则无需改数 |
| 数据明显错误或缺失必填项，需修改后重交 | 驳回 | 任务变整改中，供应商需修改数据并重新提交 |
| 仅需对方解释某计算公式或单位换算 | 提出疑点 | 不改变采集状态 |
| 要求更换/补充某类凭证（如检测报告） | 驳回 | 需重新上传并再次提交 |

### 2.3 阶段 2：计算（LCA）

| 子状态枚举 (stageSubStatus) | 界面展示文案 | 说明 |
| :--- | :--- | :--- |
| `uploaded` | 草稿待提交 | 已上传 LCA 草稿，待提交核查 |
| `submitted` | 核查中 | 已提交，核查机构处理中 |
| `clarifying` | 待澄清 | 需澄清或补充 |
| `waiting_supplier` | 供应商整改中 | 转派供应商补充 |
| `verified` | 已通过 | 核查通过，可进入下一阶段 |

### 2.4 阶段 3：核查

| 子状态枚举 (stageSubStatus) | 界面展示文案 | 说明 |
| :--- | :--- | :--- |
| `pending_review` | 待运营验收 | 待运营确认验收 |
| `rejected` | 已驳回 | 验收驳回 |
| `on_chain` | 已上链 | 验收通过，数字凭证已上链 |

### 2.5 阶段 4：报告

阶段 4 使用 **reportSubStatus**，所有报告相关操作在**报告管理**完成。

| 子状态枚举 (reportSubStatus) | 界面展示文案 | 说明 |
| :--- | :--- | :--- |
| `released` | 已下发，待供应商确认 | 已下发，等待供应商确认接收或申诉 |
| `to_archive` | 待归档 | 供应商已确认，待运营归档上链 |
| `archived` | 已归档 | 已确认归档并上链 |
| `revoked` | 已作废 | 报告/任务已作废 |
| （其他） | 待处理 | 兜底展示 |

---

## 3. 与界面、数据的对应关系

- **任务列表**：展示 5 段进度条（配置 / 采集 / 计算 / 核查 / 报告）；阶段 4 可展示「报告：xxx」子状态文案，与上表一致。
- **任务详情页**：顶部信息条第三块为「XX 状态」（配置状态 / 采集状态 / **计算状态** / 核查状态），仅展示当前任务在**当前页面阶段**下的子状态文案，数据通过 `getTaskByTaskId(taskId, pageStageIndex)` 按阶段取数，保证与列表一致且跟随页面。
- **实现约定**：子状态枚举与展示文案以本文档及 `js/mockTasks.js` 中 `getStageStatusText` 为准；新增或修改子状态时需同步更新文档与代码。
