## 三、共享样式文件

### 3.1 task_detail.css

- **适用页面**：所有使用「统一骨架」的任务详情页（operator 的 config/collect/lca/verify/certify、certifier 的 verify/certify、supplier 的 task_stage_*）。
- **主要内容**：阶段步骤条（.task-stage-steps）、任务信息仪表盘（.task-info-dashboard）、统一顶部/主体布局（.task-detail-unified__*）、可滚动主体留白（.task-detail-scroll-body）等。
- **使用约定**：
  - 凡有「吸底操作栏」的详情页，主滚动区应使用 `.task-detail-scroll-body` 或等价 `padding-bottom: 80px`（≥72px），避免最后一行被遮挡。
  - 顶部内边距统一由 `.task-detail-unified__top .task-detail-unified-top-inner { padding: 0 24px; }` 提供，各页无需再覆盖。

### 3.2 task_detail_refactor.css

- **适用页面**：采用「4 区块重构布局」的页面（如 operator/task_detail_collect_refactor_example.html）。
- **主要内容**：el-page-header、任务信息卡片（.task-info-dashboard-card）、工作台 Tabs（.task-workspace）、操作区与 TDS 吸底栏（.task-action-tds-bar）、.task-detail-scroll-body 等。
- **与 task_detail.css 关系**：refactor 为独立一套布局与类名；主用详情页以 task_detail.css + 统一骨架为主，refactor 仅示例页使用。

## 四、.task-info-dashboard 保留说明

- **保留原因**：operator 的 config/collect/lca/verify/certify 及 certifier 的 verify/certify 等页仍在使用 `.task-info-dashboard` 及其子类（如 `.task-info-dashboard__row`、`.task-info-dashboard__main`、`.dash-info-col` 等）展示任务编号、订单、供应商、产品、规格及状态。
- **约定**：在未全部迁移至 el-descriptions 或其它统一组件前，不在 task_detail.css 中删除 .task-info-dashboard 相关样式；仅在已完全改用新结构的页面（如 refactor 示例）中不再引用该类名。

## 五、脚本依赖

- **taskDetailLayout.js**：任务详情布局与角色分发（getCurrentRole、getStageSteps、initRightPanel 等），operator/certifier 多阶段详情页依赖。
- **layout.js / certifierLayout.js / supplierLayout.js**：各端顶栏、侧栏与 runXxxApp 由对应 layout 提供。
