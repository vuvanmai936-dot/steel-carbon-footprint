# 模板配置页（task_detail_config）优化说明

## 一、关键逻辑说明

### 1. 模板匹配
- **产品强关联**：模板下拉框仅展示与当前任务产品类型匹配的模板。当前 Mock 任务为 TSK-2026-892（产品：HRB400E螺纹钢），故仅展示「HRB400E螺纹钢碳核算模板」一条选项。
- **占位提示**：未选择时显示「请选择对应产品的核算模板」。
- **扩展方式**：在页面脚本内 `PRODUCT_TEMPLATES` 数组中增加项即可；正式环境应由接口按 `taskInfo.productName` / 产品类型过滤返回。

### 2. 状态流转
- **保存草稿**：仅向协作日志追加一条记录（时间、操作人、动作「保存模板配置草稿」），不改变任务阶段与状态文案。
- **确认配置并下发**：
  1. 校验：未选择模板时弹出 Element Plus 警告「请先选择与HRB400E螺纹钢匹配的核算模板」；
  2. 二次确认：弹框「确认配置并下发任务」；
  3. 追加协作日志：如「09:30 Admin_001 配置模板并下发至沙钢集团，任务进入采集阶段」；
  4. 本地状态：`configIssued = true`，任务状态文案变为「模板已配置，待供应商填报」；
  5. 进度条：「采集」阶段由置灰变为待激活（浅蓝 `is-pending-activate`）；
  6. Mock 持久化：若存在全局 `saveTaskSnapshot`，则把当前 Spread 的 JSON 写入该任务 Snapshot，供采集页/供应商填报使用。

### 3. 按钮交互
- **保存草稿**：次要按钮，点击即写日志并提示「草稿已保存」，不校验是否已选模板。
- **确认配置并下发**：主要按钮；未选模板时禁用或点击时提示上述警告；已选模板且 Spread 已加载时可点，弹框确认后执行上述状态与日志更新。

## 二、原型测试注意事项

### 1. Mock 数据修改方式
- **任务信息**：默认使用 URL 参数 `taskId=TSK-2026-892`；无参数时使用脚本内 `defaultTaskId = 'TSK-2026-892'`。修改任务可改 `operator/task_detail_config.html` 中 `defaultTaskId` 或在地址栏加 `?taskId=TSK-2026-xxx`（需在 `js/mockTasks.js` 的 `MOCK_TASK_MAP` 中存在）。
- **模板列表**：在 `PRODUCT_TEMPLATES` 中增删改，当前仅一条「HRB400E螺纹钢碳核算模板」。
- **协作日志**：初始两条在 `configLogs` 中，第一条为系统创建，第二条为空行；后续由「保存草稿」「确认配置并下发」追加。
- **核查机构 / 核算周期**：`selectedCertifier`、`accountingPeriod` 在 data 中，核算周期默认「2026-01至2026-03」，可按需改默认值。

### 2. 页面跳转规则
- **返回按钮**：回到任务列表 `self_operated_task_list.html`（若 URL 带 `from=order` 则回 `order.html`）。
- **面包屑**：「中心运营端」「任务管理」点击均跳转任务列表 `self_operated_task_list.html`（可带当前 `taskId` 查询参数）；「模板配置」为当前页不跳转。
- **阶段进度条**：当前为「配置」页，点击「采集」「计算」「核查」「报告」会跳转至对应阶段页（由 `taskDetailLayout.getStageSteps` 生成的 `href`，如 `operator/task_detail_collect.html?taskId=xxx`）。注意：从配置页点击「采集」时，若未先在本页执行「确认配置并下发」，采集页仍可打开，但业务上应以先下发的状态为准。

### 3. 其他
- **SpreadJS**：需在 Chrome 等现代浏览器中打开，CDN 加载 16.x；评估版水印通过页面内 CSS 隐藏（`.gc-watermark` 等），正式授权版可去掉该样式。
- **分辨率**：样式按 1920×1080 适配，表格与协作日志区可纵向滚动、自适应高度。
