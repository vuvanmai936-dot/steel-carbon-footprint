# 澄清区域「钉钉式对话」设计方案

本文档为澄清区域提供**类似钉钉对话**的界面与交互设计，便于产品与开发评审。目标：让澄清沟通更直观、易用，贴近日常 IM 习惯。

---

## 1. 设计目标与对照

| 目标 | 说明 |
| --- | --- |
| **对话感** | 消息以气泡形式呈现，自己与对方左右分列，一目了然 |
| **结构清晰** | 主题列表 = 会话列表；选中主题 = 打开该会话；底部输入 = 回复 |
| **信息密度适中** | 头像/角色、时间、未读、附件等在有限空间内合理呈现 |
| **操作顺手** | 输入框常驻底部、发送即显、新建主题入口明确 |

与钉钉对话的对照：

| 钉钉 | 澄清区域对应 |
| --- | --- |
| 左侧会话列表 | 澄清主题列表（含标题、最后消息摘要、时间、状态） |
| 右侧消息区 | 选中主题的消息区（气泡式时间线） |
| 底部输入框 | 回复输入框 + 发送 |
| 新建会话 | 新建澄清（发起新主题） |
| 单聊/群聊 | 单主题多角色（运营/供应商/核查机构） |

## 3. 澄清主题列表（类似钉钉会话列表）

### 3.1 单条展示内容

| 元素 | 说明 | 样式建议 |
| --- | --- | --- |
| 头像/标识 | 角色图标或首字母（运营 O / 供应商 S / 核查 V） | 小圆，按角色配色 |
| 主题标题 | 澄清主题（subject） | 加粗，最多一行截断 |
| 最后消息摘要 | 最后一条消息内容前 20 字 | 灰色小字，单行截断 |
| 时间 | 最后更新时间（相对：刚刚/5 分钟前/昨天） | 右对齐，小字 |
| 未读/状态 | 待回复红点、进行中/已关闭标签 | 红点或小角标 |
| 来源标签 | 「来自核查机构」高亮 | 红色边框或背景 |

### 3.2 列表交互

- 点击主题 → 进入该主题消息区，可选标记已读
- 「+ 新建澄清」→ 展开内嵌表单（与现有一致），提交后自动选中新主题
- 支持按状态筛选：全部 / 进行中 / 已关闭（可选）
- Drawer 宽度 &lt; 400px 时，可仅展示消息区，列表收为顶部下拉选择

## 4. 消息区（钉钉式气泡布局）

### 4.1 消息气泡规则

- **自己发的**（当前用户角色）：气泡靠右，主色（如蓝）
- **他人发的**：气泡靠左，浅灰底
- **同一人连续多条**：可合并显示，只在第一条显示头像与角色，后续仅气泡 + 时间（可选）

### 4.2 单条消息结构

```
┌──────────────────────────────────────────┐
│ [头像] 角色名 · 时间                      │  ← 靠左
│ ┌────────────────────────────────────┐   │
│ │ 消息内容（支持换行、链接、必要时富文本）│   │
│ └────────────────────────────────────┘   │
└──────────────────────────────────────────┘

         ┌────────────────────────────────────┐
         │ 消息内容                            │  ← 靠右（自己）
         └────────────────────────────────────┘
         [头像] 运营 · 11:30
```

### 4.3 气泡样式建议

| 自己 | 他人 |
| --- | --- |
| 背景 #ECF5FF，边框 #B3D8FF | 背景 #F5F7FA，边框 #E4E7ED |
| 圆角 8px，右下角略尖（可选） | 圆角 8px，左下角略尖 |
| 最大宽度 75%，右侧留白 | 最大宽度 75%，左侧留白 |

### 4.4 消息内容扩展

- **纯文本**：当前已有，保持
- **附件**：若有 `attachments`，在气泡下方展示文件名 + 图标，点击预览/下载
- **引用回复**：可选，引用某条消息时在气泡上方显示「引用：xxx...」

### 4.5 时间展示

- 同一天：10:30、14:20
- 跨天：昨天 10:30、3月1日 10:30
- 分组：超过 5 分钟可插入分组线「—— 14:20 ——」

## 5. 底部输入区（常驻）

### 5.1 布局

```
┌─────────────────────────────────────────────────────────┐
│ ┌───────────────────────────────────────────────────┐   │
│ │ 输入回复内容...                                    │   │
│ │ （支持多行，2～4 行）                              │   │
│ └───────────────────────────────────────────────────┘   │
│ [附件] [@提及] (可选)                        [发送]     │
└─────────────────────────────────────────────────────────┘
```

- 多行输入框（2～4 行可调）
- 发送按钮在右下
- 已关闭的澄清：输入框置灰，提示「该澄清已关闭」

### 5.2 可选能力（后期迭代）

- 附件上传：图片、PDF 等
- @ 提及：@运营 / @供应商 / @核查机构
- Enter 发送、Shift+Enter 换行（可配置）

---

## 6. 新建澄清（内嵌在侧栏）

保持当前「内嵌表单」方式，不再弹窗：

- 点击「+ 新建澄清」→ 列表上方展开表单区块
- 字段：主题类型、标题、首条内容
- 提交后：创建会话，自动选中并跳转到该主题消息区，首条内容作为第一条消息显示

---

## 7. 数据与展示约定

### 7.1 与现有 mock 的对应

- `clarificationList` → 主题列表（每条含 lastMessage、updatedAt）
- `selectedClarDetail.messages` → 消息区气泡列表
- `replyContent` + `submitClarifyReply` → 底部输入与发送

### 7.2 前端需补充（可选）

- 消息 `attachments: []` 结构，用于附件展示
- 相对时间格式化函数
- 当前用户角色判断，用于气泡左右对齐

## 8. 响应式与窄屏

| Drawer 宽度 | 建议 |
| --- | --- |
| ≥ 500px | 左右分栏：列表 + 消息区 |
| 400～500px | 列表收窄，消息区为主 |
| &lt; 400px | 单栏：顶部为主题选择下拉，下方为消息区 + 输入 |

## 9. 实现优先级建议

| 优先级 | 内容 |
| --- | --- |
| P0 | 消息区气泡式布局（自己/他人左右分列） |
| P0 | 底部常驻输入框 |
| P0 | 主题列表展示最后消息摘要与时间 |
| P1 | 主题列表头像/角色标识、未读红点 |
| P1 | 时间相对展示、消息分组线 |
| P2 | 附件展示、@ 提及、引用回复 |

---

## 10. 参考示意（钉钉对话）

钉钉单聊/群聊特点：

- 会话列表：头像 + 名称 + 最后一句 + 时间 + 未读数
- 消息区：气泡、左右分列、时间分组
- 底部：输入框 + 发送 + 附件/表情等

本方案将「澄清主题」映射为「会话」，「消息」映射为「聊天消息」，交互与钉钉保持一致，以降低用户学习成本。
