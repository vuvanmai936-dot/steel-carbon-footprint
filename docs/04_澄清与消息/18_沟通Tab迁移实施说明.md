# 沟通 Tab 迁移实施说明

本文档说明「消息与澄清」迁移为任务详情主内容区「沟通」Tab 的已落地方案，供判断是否合适及后续迭代参考。

---

## 一、方案结论

- **结论**：已采纳并实施。任务详情主内容区采用 Tab 时，**最后一项固定为「沟通」Tab**。沟通区采用**单一日志流（RFI Log）**形式：无左侧主题列表，整区为一条按时间排序的沟通记录流；**协作日志**以系统消息卡片形式与人工消息按时间混排在同一流中；底部统一输入「输入澄清问题...」+ 附件 + 发送；标题区为「澄清与沟通记录 (RFI Log)」+「导出记录 (PDF)」。不单独设置澄清议题标题，所有沟通内容在同一对话框内处理。支持附件/图片的上传与下载。入口统一、易发现，与核查端参考设计一致。
- **命名**：全站统一为「沟通」；沟通区标题为「澄清与沟通记录 (RFI Log)」。

---

## 二、已实施页面与改动要点

|------|-----------------|---------------|------------|

| 运营 verify | 核算报告 \| 核查报告 \| 核查声明 \| **沟通** | 单一日志流：标题 + 导出 PDF + 任务动态 + 全部消息 + 统一输入（含附件） | 「沟通」→ 切到沟通 Tab |
| 核查 task_detail | 证照与报告 \| **沟通** | 单一日志流：标题 + 导出 PDF + 全部消息 + 统一输入（含附件） | 无单独按钮，由 Tab 进入 |

- 沟通 Tab 内**无左侧主题列表**，采用「澄清与沟通记录 (RFI Log)」单一日志流：协作日志（任务动态）以系统消息卡片与人工消息按时间混排；底部统一输入框「输入澄清问题...」+ 附件按钮 + 发送；新消息写入默认 RFI 会话（不单独设澄清标题）。**导出记录 (PDF)** 使用浏览器打印（可另存为 PDF）。

- 任务条原「消息与澄清」文案已改为「**沟通**」，点击行为改为切换主区到「沟通」Tab（`activeTab` / `mainActiveTab = 'communicate'`）。

## 三、深度链接与消息中心

- **URL**：支持 `?open=clarify` 或 `?tab=communicate`，进入页时默认切到「沟通」Tab。
- **消息中心**：顶部「消息」铃铛打开消息中心 Drawer；从消息中心跳转至某任务时，建议使用 `?taskId=xxx&open=clarify`，进入该任务详情并自动打开「沟通」Tab。详见 [12_任务详情页统一实施方案](../03_任务详情/12_任务详情页统一实施方案.md) 1.4 与顶部消息铃铛说明。

## 四、样式与逻辑复用

- 沟通 Tab 内沿用 `css/clarify_panel.css` 及 [17_澄清区域钉钉式对话实施方案](17_澄清区域钉钉式对话实施方案.md)。单一日志流使用 `.rfi-log-header`、`.rfi-log-main`、`.rfi-log-messages`；协作日志以**系统消息卡片**（`.clarify-log-card`）与人工消息气泡（`.clarify-msg`、`.clarify-msg-bubble`）按时间混排；消息支持 `attachments[]`，气泡内用 `.clarify-msg-attachments`、`.clarify-msg-attachment-item` 展示附件链接；底部输入区支持 `.clarify-input-with-attach`、`.clarify-input-attach-list` 附件列表。Mock 提供 `getDefaultRfiSessionId(taskId)`，新消息通过 `replyClarification(taskId, defaultRfiSessionId, { content, attachments })` 写入默认会话。导出 PDF 为 `window.print()`。
- 数据：`rfiLogMessages` 计算属性合并当前页协作日志（events/configLogEvents/logEventsForDrawer）与 `clarificationList` 下全部会话的 messages，按 `createdAt` 排序后渲染；澄清加载时机为「当前 Tab 为沟通」或 `mainActiveTab === 'communicate'`。

## 五、是否合适的自检

| 问题 | 结论 |
|------|------|
| 是否将「消息与澄清」移到主内容区下方、并作为最后 Tab？ | **是**，已统一为「沟通」Tab。 |
| 是否始终放在最后边的 Tab？ | **是**，所有任务详情页主内容 Tab 的最后一项均为「沟通」。 |
| 是否更名为「沟通」？ | **是**；核查端可用「沟通 (RFI)」作副标题。 |
| 方案是否合适？ | **合适**：入口统一、减少 Drawer 依赖、对齐参考设计，且不损失现有功能。 |
