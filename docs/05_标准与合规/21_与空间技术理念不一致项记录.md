# 与空间技术理念不一致项记录

本文档用于记录在**产品设计、交互或实现**中发现的、与**可信数据空间 / 数据基础设施**技术理念或相关标准不一致的项，便于后续修正与合规对齐。  
对照依据见 [20_可信数据空间标准索引与理念摘要.md](20_可信数据空间标准索引与理念摘要.md)。

---

## 使用说明

- **何时登记**：在设计评审、实现审查或需求讨论中，一旦发现某功能/交互/文案/接口与「身份管理、连接器、目录、数字合约、使用控制」或「统一目录标识、统一身份登记、统一接口要求」等理念不符，即在本表新增一行。  
- **谁可更新**：产品、设计、开发及参与评审的各方均可补充；建议由发现人填写「发现位置」与「不一致描述」，由技术/合规侧补充「建议修正」与「参考标准」。  
- **状态**：`待评估` → `已采纳（待实施）` / `已实施` / `不予采纳（说明原因）`。
