# 模板与标准中心逻辑说明

## 1. 业务目标
将非结构化的 Excel 文件转化为系统可识别的结构化数据配置（Schema），并定义供应商的数据提交流程。

## 2. Excel 解析逻辑 (前端/后端)
当用户上传 `.xlsx` 文件时，系统需执行以下解析：
1. **Sheet 识别**: 每个 Sheet 映射为一个“工序 (Process)”。
2. **Header 识别**: 默认读取第一行作为“字段名”。
3. **数据清洗**: 忽略空行、忽略隐藏 Sheet。

## 3. 凭证配置逻辑 (Evidence Configuration)
* **凭证与工作簿绑定**：凭证要求与整个工作簿（Workbook）绑定，不做行级关联。供应商需按凭证清单整体提交，不再配置「关联数据行」。
* **数据结构**（已废弃 `relatedFields`）:
  ```json
  {
    "requirementId": "req_001",
    "name": "电力发票",
    "description": "需盖章...",
    "exampleFileUrl": "http://..."
  }
  ```
* **废弃说明**：`relatedFields` 行级关联已废弃。原设计中可将凭证关联到具体数据行，现统一为与工作簿绑定；凭证清单由 Snapshot 的 `evidenceRequirements` 维护，通过 taskId 与任务关联。
