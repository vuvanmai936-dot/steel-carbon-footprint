## 维护规则

- **只追加，不在历史上"返工"**：发现错误请在现行文档（被取代方）处更正，不要回写本目录。
- **不在主路径引用本目录**：除「这里有历史背景」之类的脚注外，所有功能、状态、字段定义请引用现行 SoT。
- **新增归档**：若有方案被新规则吸收，请在 [`docs/00_迭代计划/设计变更记录.md`](../00_迭代计划/设计变更记录.md) 写一条记录，再 `git mv` 到本目录并在文件首行加「归档说明」。
