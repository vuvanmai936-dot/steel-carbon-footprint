# 招采参考清单版式规格（用于碳足迹对齐）

基于 `reference_formats/招采-工作量评估与报价清单.xlsx` 解析结果。

## 工作表

- **单工作表**，名称示例：`招采电子交易平台分布式拆分` → 碳足迹侧用 `碳足迹数据服务系统`。
- **无独立封面 sheet**，表前可用一行合并标题（荣泽侧实现为第 1 行）。

## 表头（第 1 行为数据表头，若上方另有标题行则顺移）

| 列 | 标题 |
|----|------|
| A | 子模块 |
| B | 一级功能 |
| C | 二级功能 |
| D | 预估工作量(人天) |
| E | 单价 |
| F | 预估报价(元) |

## 数据行约定

- **子模块**：大块业务域；同一块内 A 列纵向合并。
- **一级功能**：子模块下的功能分组；同一子模块内相同「一级功能」B 列纵向合并。
- **二级功能**：可报价的细项描述（对应碳足迹 `DETAIL_ROWS` 的「功能点」，二阶段/可选在文案中标注阶段）。
- **工作量**：人天，整数（碳足迹与招采一致为人天；招采历史表曾有人月截图，以当前参考 xlsx 为准）。
- **单价**：人天行统一 **1500**；第三方 SpreadJS 行无「人天」时，D/E 留空，**F 直接填固定金额**（不按 D×E）。
- **预估报价**：人天行 **= D×E**；固定价行 **= 授权费**。

## 合计行

- A–C 列合并，正文 **总计**。
- D 列：`SUM` 人天列（不含 SpreadJS 空单元格）。
- F 列：`SUM` 金额列（含固定价）。
- 参考表 E 列合计行为单价占位（1500）；碳足迹合计行 E 可留空或填 **1500** 仅作说明。

## 编制说明（表下）

- 说明人天单价、SpreadJS 固定费、总价闭合方式及与招采模板列名一致。
